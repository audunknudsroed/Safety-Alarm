Safety-alarm

Android application which alerts a second application user if the first user does not come back home in time.

Goal

Use mobile phone technology to reduce parental anxiety

Idea

To create an Android application which to users can install on their phones and use to verify that one of them or both safely make it to some pre-decided destination.

The two users agree beforehand to where the subject user should arrive, and when. The arrival is detected by the user's contact with a WiFi network SSID at the destination. The other user is then notified of this arrival via some communication technology, for example synchronization to a server or via SMS.

The receival of such a notification causes the receivers application to NOT give any alert. Failure to receive the notification within the time-limit does cause an alert.

Overview of Intended Incremental Progress

  1. Two users can agree to one of the users waiting for the arrival of the other.
  2. Multiple users can set up checks on each other.
  3. Location tracking, either via GPS or identification of cellular base stations.

Challenges

The main challenge is that it's most critical function (checking in at the safe destination) will often be done at the end of the day, when the phone battery energy level is likely to be very low. Care must be taken to make the application consume as little energy as possible, so that the application does not prevent it self from working by depleting the battery before the user comes home.
